# Security Policy

## Responsible Disclosure
Please do not publicly disclose vulnerabilities before contacting us and allowing time for a fix.

## Reporting a Vulnerability

* **For low to medium severity vulnerabilities:**
  * Check if the issue is already fixed in a newer version.
  * Look for existing issues, pull requests, or branches addressing the problem.
  * If not already reported, you may open a public issue (note: this will be visible to everyone, so only do this for low-risk issues), or contact us privately at [paw.all.obo.alexaro@outlook.com](mailto:paw.all.obo.alexaro@outlook.com).

* **For critical vulnerabilities:**
  * Contact us immediately and privately at [paw.all.obo.alexaro@outlook.com](mailto:paw.all.obo.alexaro@outlook.com).  
  * Please do not open a public issue for critical vulnerabilities.

If possible, please provide steps to reproduce the issue and any relevant details. We appreciate your help in making this project more secure.
